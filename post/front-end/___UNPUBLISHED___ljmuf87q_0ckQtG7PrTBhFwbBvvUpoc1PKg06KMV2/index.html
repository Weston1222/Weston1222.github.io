ngOnChanges

當元件內有@Input()時會在初始化時會先執行 ngOnChanges()方法再執行 ngOnInit()方法

每當＠Input()有變化時都會呼叫 ngOnChanges()方法(參考位置(reference)有變才會呼叫)

ngOnChanges()可以放一個 SimpleChanges(key/value) 型別的參數，key 是 @Input()的名字，valus 是 SimpleChanges 型別

import { OnChanges, SimpleChanges } from "@angular/core";
export class AppComponent implements OnChanges {
  @Input() item;
  ngOnInit() {
    // do something...
  }
  ngOnChanges(sc: SimpleChanges) {

    //firstChange：boolean 值，第一次呼叫時為 true，之後都是 false
    console.log(sc["item"].firstChange;)

    //previousValue：上次呼叫時的值
    console.log(sc["item"].previousValue;)

    //currentValue：這次變更時的值
    console.log(sc["item"].currentValue;)

  }
}

ngDoCheck

當@Input()是一個物件，因為沒有改變參考位置(reference)所以不會呼叫上面提到的 ngOnChanges 方法，此時可以使用 ngDoCheck 來自行判斷

當遇到以下四種情況會觸發 Angular Change Detection 變化檢測策略

  1. 組件的@Input() 引用發生變化。
  2. 組件的 DOM 事件，包括它子組件的 DOM 事件，比如 click、submit、mouse down 等事件。
  3. Observable 訂閱事件，同時設置 Async pipe。
  4. ChangeDetectorRef.detectChanges()、ChangeDetectorRef.markForCheck()、ApplicationRef.tick()，手動調用這三種方式觸發變化檢測。

attribute directive

透過 attribute directive 可以擴充元件的屬性，

ng g d name

CLI 生成的檔案

import { Directive } from "@angular/core";

@Directive({
  selector: "[appName]",
})
export class NameDirective {
  constructor() {}
}

directive 都要有@Directive妝飾器

selector: "[appName]"代表套用的屬性名稱 比如說<button appName=""></button>

selector: button[appBsButton]代表只有 button 能用

先注入 ElementRef 來取得 directive 所在的 element，並在元件初始化時套用

<button appName="">Button</button>
import { ElementRef, Directive, OnInit } from "@angular/core";

@Directive({
  selector: "[appName]",
})
export class NameDirective implements OnInit {
  constructor(private el: ElementRef) {}

  ngOnInit() {
    const button = this.el.nativeElement as HTMLElement;
    button.classList.add("btn");
    button.classList.add("btn-primary");
  }
}

等同於

<button class="btn btn-primary">Button with Attribute Directive</button>

Renderer

注入 Renderer，renderer.setElementClass()的第 3 個參數代表要不要加入這個 class

import { Renderer, ElementRef, Directive, OnInit } from "@angular/core";

@Directive({
  selector: "[appName]",
})
export class NameDirective implements OnInit {
  constructor(private el: ElementRef, private renderer: Renderer) {}

  ngOnInit() {
    this.renderer.setElementClass(this.el.nativeElement, "btn", true);
    this.renderer.setElementClass(this.el.nativeElement, "btn-primary", true);
  }
}

使用@Input

使用@Input 可以接收 template 傳進來的參數

export class NameDirective implements OnInit {
  @Input() appName;

  constructor(private el: ElementRef, private renderer: Renderer) {}

  ngOnInit() {
    this.renderer.setElementClass(this.el.nativeElement, "btn", true);
    this.renderer.setElementClass(this.el.nativeElement, `btn-${this.appName || "primary"}`, true);
  }
}

@HostListener 與 HostBinding

@HostListener 監聽一個 DOM 事件，並在 Element 被消滅時自動取消監聽

@HostBinding 綁定一個 DOM 屬性，如果@ostBinding 有變化，被綁定的 DOM 屬性也會被更新

@Directive({
  selector: "[appRainbow]",
})
export class RainbowInputDirective {
  public possibleColors = ["darksalmon", "hotpink", "lightskyblue", "goldenrod", "peachpuff", "mediumspringgreen", "cornflowerblue", "blanchedalmond", "lightslategrey"];
  //字體颜色
  @HostBinding("style.color") color: string;
  //邊框颜色
  @HostBinding("style.borderColor") borderColor: string;
  //監聽鍵盤事件
  @HostListener("keydown") onKeyDown() {
    //隨機賦予顏色
    const index = Math.floor(Math.random() * this.possibleColors.length);
    this.color = this.borderColor = this.possibleColors[index];
  }
}

@Component({
  selector: "app-rainbow-input-demo",
  template: `
    <h3>這是一个彩虹輸入框，每輸入一次都會改變颜色</h3>
    <input [class]="'my-input'" type="text" appRainbow />
  `,
  styles: [
    //省略
  ],
})
export class RainbowInputDemoComponent {}

exportAs

在宣告@Directive()設定一個exportAs: string，用來代表@Directive()的實體

可以把@Directive()丟到 Component 去使用

@Directive({
  selector: '[appColorful]',
  // 加上 exportAs
  exportAs: 'colorful'
})
@Component({
  selector: "my-app",
  template: `
    <!-- 使用 #color="colorful" 取得實體 -->
    <p appColorful="blue" #color="colorful">Hello World</p>
    <button (click)="change()">Change Color</button>
  `,
  styleUrls: ["./app.component.css"],
})
export class AppComponent {
  // 確定取得的是 ColorDirective 的實體
  @ViewChild("color") color: ColorfulDirective;

  change() {
    console.log(this.color);
    this.color.changeColor("black");
  }
}

HTTP_INTERCEPTORS

HTTP_INTERCEPTORS 用來攔截 Http Request，可以將要發送出去的 Request 加工之後再發送出去

先建立一個 service，並實作 HttpInterceptor 介面

@Injectable()
export class TokenAuthHttpInterceptor implements HttpInterceptor {
  constructor() {}
  public intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    // token 可以來自任何地方
    const newRequest = req.clone({
      setHeaders: { Authorization: "Bearer 123456" },
    });
    return next.handle(newRequest);
  }
}

req: HttpRequest<any>代表目前送出的 HttpRequest

使用req.clone()建立新的 Request

next: HttpHandler將處理完的 HttpRequest 放入handle方法裡，return 出去

在@NgModule裡設定providers

@NgModule({
  ...,
  providers: [{
    provide: HTTP_INTERCEPTORS,
    useClass: AppHttpInterceptor,
    multi: true,
  }],
})
export class AppModule {}

@NgModule 的 providers

@NgModule 中的 providers用來設定依賴注入

{
  provide: SomeClass,
  useXXXX: YYYY
}

provide代表要注入的類

useClass幫要注入的類取一個名字，並產生實體，如果 serive 要建立多實體就取多個名字

@NgModule({
  providers: [
    SomeClass
  ]
})

// 和以下程式完全一樣意思
@NgModule({
  providers: [
    {
      provide: SomeClass,
      useClass: SomeClass
    }
  ]
})

以下寫法為當構造函數請求 Class1 時，Angular DI 創建 Class2 的實例並將其傳遞給構造函數，用於當 serive 需要取代另一個 serive 時(會產生兩個實體)

providers: [
  Class2,
  {
    provide: Class1,
    useClass: Class2,
  },
];

useExisting

如果要共用一個實體則需要使用useExisting，當構造函數需要 Class1 時，Angular DI 從另一個 key 為 Class2 的 provider 裡查找，取出對應的實例進行注入(只會有一個實體)

providers: [
  Class2,
  {
    provide: Class1,
    useExisting: Class2,
  },
];

useValue

使用useValue將 serive 裡的某個物件替換

@Injectable({
  providedIn: "root",
})
export class DataService {

    return 100;

}

建立要替換的物件

const newData = {
  getData: () => {

用下面的方法替換

@NgModule({

    {
      provide: DataService,

OnPush 和 markForCheck

在@Component內增加changeDetection: ChangeDetectionStrategy.OnPush

當設定為 OnPush 時，只有在元件的 @Input 變更，且真正有變更時(變更參考位置)，才會進行變更偵測

  template: `
    Num: {{ data.num }}
  `,
  changeDetection: ChangeDetectionStrategy.OnPush
})
