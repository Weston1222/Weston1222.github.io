Axios 特性

Axios 用在瀏覽器端會使用 XMLHttpRequests，如果在 Node.js 中則會使用 http 模組。

返回的是 Promise 物件， 預設使用 JSON 格式傳送。

載入 Axios

cdn

<script src="https://unpkg.com/axios/dist/axios.min.js"></script>

npm

npm install axios

import

import axios from 'axios';

基本語法

axios 回傳的是 Promise 物件。

//默認為GET請求
const example = axios('url');
example
  .then((response) => console.log(response))
  .catch((err) => console.log(err));

通常用axios.get()或axios.post()等等。

GET 參數

axios.get()函數的第二個參數可以放入一個對象，並在裡面放入params參數，等同於在 url 後面加上參數和參數值。

axios
  .get('url', {
    params: {
      name: '小明', //   url/?name=小明&id=123
      id: '123',
    },
  })
  .then((response) => console.lig(response))
  .catch((err) => console.log(err));

可搭配Promise.all()發送多個請求。

function getOne() {
  return axios.get('url', {
    params: {
      name: '小明', //   url/?name=小明&id=123
      id: '123',
    },
  });
}

function getTwo() {
  return axios.get('url', {
    params: {
      name: '小王', //   url/?name=小明&id=123
      id: '456',
    },
  });
}

Promise.all([getOne(), getTwo()]).then((response) => {
  const userOne = response[0];
  const userTwo = response[1];
  console.log(userOne);
  console.log(userTwo);
});

POST 參數

axios
  .post('url', {
    email: 'abc@gmail.com',
    password: '1234',
  })
  .then((response) => console.log(response))
  .catch((error) => console.log(error));

config 物件

可以把 method、url、data 等等放在一個物件裡。

axios({
  method: 'post', //POST請求
  url: 'url',
  data: {
    //  data要傳給API的資料
    id: '小明',
    password: '123456',
  },
}).then((response) => console.log(response));

create 預設 config 物件

使用axios.create()建立預設資料。

const exampleConfig = axios.create({
  baseURL: 'url',
  timeout: 1000,
  headers: { 'X-Custom-Header': 'foobar' },
});

baseURL：預設 url，會加在後面設定的 url 的前面。 headers：發出請求的表頭。 timeout：請求時間超過這個時間的話，中斷請求。

const data = {
  id: '小明',
  password: '123456',
};
const exampleApi = (data) => exampleConfig.post('url', data);

exampleApi(data)
  .then((response) => console.log(response))
  .catch((err) => console.log(err));

回傳結構

{
  "data": {},
  "status": 200,
  "statusText": "OK",
  "headers": {},
  "config": {},
  "request": {}
}

Interceptors 攔截器

攔截器可以再發出 request 或是接收 response 之前做一些處理。

request

先用create()函式新建一個實例。

const exampleConfig = axios.create({
  baseURL: 'url',
  timeout: 1000,
  headers: { 'X-Custom-Header': 'foobar' },
});

創建好實例之後使用interceptors.request.use()方法來製作攔截器。

interceptors.request.use()需要放兩個函式做為參數：

第一個函式可以在送出 request 前對 config 做一些修改。 第二個則是錯誤處理。

exampleConfig.interceptors.request.use(
  function (config) {
    // Do something
    return config;
  },
  function (error) {
    // Do something
    return Promise.reject(error);
  }
);

response

和 request 很像，使用axios.interceptors.response.use()函式設定。

exampleConfig.interceptors.response.use(
  function (response) {
    // 可以對response做一些修改
    return response;
  },
  function (error) {
    if (error.response) {
      switch (error.response.status) {
        case 404:
          // go to 404 page
          break;
        case 500:
          // go to 500 page
          break;
        default:
          console.log(error.message);
      }
    }
    if (!window.navigator.onLine) {
      alert('網路有問題');
      return;
    }
    return Promise.reject(error);
  }
);

設定完攔截器之後使用exampleConfig.post()送出請求就可以了。
