Hexo 筆記

參考自 試著學 Hexo

環境 安裝 Hexo CLI

  1. 安裝 npm（略過） 檢查是否安裝

    node -v
npm -v

  2. 安裝 Hexo

    npm install -g hexo-cli

  3. 檢查

    hexo -v

建立 Hexo

terminal 移動到想要放的位置

初始化

hexo init [資料夾名稱]

Hexo 目錄結構

+-- node_modules
+-- scaffolds
+-- source
|   +--_posts
+--themes
|   +--landscape
+-- _config.yml
+-- .gitignore
+-- package.json
+-- package-lock.json

node_modules

Hexom 的套件和之後安裝的套件會放在這

scaffolds

scaffolds 底下會有三個檔案

  • draft.md
  • page.md
  • post.md 會以這裡的檔案初始化新的頁面

souecs

_post 主要用來放原始檔的地方 如果有不想被編譯得資料夾可以使用_下底線開頭，(_post)不算。

themes

themes 用來放 hexo 主題的地方，Hexo 會依據該資料夾下的主題與_config.yml 設置產生靜態檔案。

_config.yml

Hexo 設定檔，用來設定主題、網誌名稱等 與上面主題_config.yml 檔不同

.gitignore

不想加入版控的檔案寫在這裡

package.json

管理 npm 下載的檔案

package-lock.json

npm 套件管理檔

HexoCLI 常用指令

新建文章

hexo new '文章名'

新建的文章會放在 source/_post

hexo new page about

在 source/about 新建 about page

開啟模擬伺服器

hexo server

開啟後瀏覽器進入http://localhost:4000 關閉按 CTRL+C

清除暫存檔

hexo clean

清除快取檔案 db.json 和編譯檔案 public

生成靜態網頁

hexo generate

根據 source/_post 的原始檔編譯靜態文件，騙譯完後會放在 public 資料夾裡。

部屬至 GitHub Pages(須先安裝套件)

hexo deploy

或是使用快速部署

hexo d -g

上面代表生成靜態網頁並部署

安裝套件

本機安裝

裝在電腦裡，可直接使用 terminal 呼叫的套件 範例：

npm install -g hexo-cli

-g 代表 global

專案安裝

將套件裝在當前資料夾

範例：

npm install hexo-generator-feed --save

--save 代表將套件資訊紀錄在 package.json

dependencies(依賴套件)

dependencies 套件是專案 production or build 後繼續使用的套件，例如：jQuery

devDependencies(開發依賴套件)

devDependencies 套件是在開發中使用的套件，不會被 Hexo 編譯 範例：

npm install --save-dev hexo-generator-feed

多了-dev

移除套件

npm uninstall [套件名稱]

本機套件加-g

npm uninstall -g [套件名稱]

還原專案套件

npm install

從 package.json 的紀錄中下載套件

Hexo 部署 GitHub pages

安裝 GitHub pages 套件

npm install --save hexo-deployer-git

安裝完後到_config.yml 加上

deploy:
  type: git
  repo: https://github.com/你的帳號/你的帳號.github.io.git
  branch: master
  message: ""

第 16 行 url 改成

url: https://你的帳號.github.io

部署 Hexo

hexo generate
hexo deploy
or
hexo d -g

去除模板

.md 檔直接改成.html 檔 在最上面加上

---
layout: false
---

Hexo 主題

下載

git clone https://github.com/theme-next/hexo-theme-next themes/next

將 GitHub 上的檔案下載到 themes/next

切換主題

到 hexo 的_config.yml 設置主題 把theme:改成theme: next

NexT 設定

設定 theme/next/_config.yml 檔

Scheme Settings 設定風格 darkmode: true 開啟暗色模式 menu 菜單設定

閱讀全文 More

  1. 手動設定 直接在要截斷的地方加上

  2. 自動截斷 在themes/_config.yml尾端加上 (需額外安裝套件 hexo-excerpt)

    auto_excerpt:
enable: true
length: 100

  3. description 在文章區塊加上

    ---
title: Hexo 重點整理
date: 2222-01-01 00:00:00
tags:
description:
---

    會顯示 description 的內容

## 這是一個不存在的頁面 很抱歉，你目前存取的頁面並不存在。 預計將在約
<span id="timeout">5</span> 秒後返回首頁。 如果你很急著想看文章，你可以 **[點這裡](URL)** 返回首頁。

<script>
  let countTime = 5;

  function count() {
    document.getElementById("timeout").textContent = countTime;
    countTime -= 1;
    if (countTime === 0) {
      location.href = "https://hsiangfeng.github.io/"; // 記得改成自己網址 Url
    }
    setTimeout(() => {
      count();
    }, 1000);
  }

</script>
